# 真鲜速达 (ZenFresh) 小程序开发大纲

## 一、项目定位

**一句话描述**：农产品中间商管理工具，连接农户与酒店，记录采购、加工、配送全流程。

**核心目标**：替代纸笔记账，实现订单可追溯、对账有依据。

---

## 二、用户角色

| 角色 | 权限 | 进入方式 |
|:---|:---|:---|
| **处理者** | 全功能：采购、销售、库存、数据 | 白名单授权 |
| **酒店** | 查看本店订单、扫码确认收货 | 绑定码激活 |
| **农户** | 查看本人供货记录 | 绑定码激活 |

---

## 三、核心数据模型

```
农户 Supplier：id, name, phone, address
酒店 Customer：id, name, contact, address, deliveryTime, notes
品类 Category：id, name, unit, lossRate

采购单 Purchase：id, date, supplierId, items[], totalAmount, status
销售单 Sale：id, date, customerId, deliveryDate, items[], status, qrCode

用户 User：openid, role, bindId, name
```

---

## 四、页面结构

```
启动页 → 登录页（三角色入口）→ 各角色首页

处理者端：
├── 首页（快捷入口 + 待办列表）
├── 采购录入
├── 销售录入 → 生成送货单（含二维码）
├── 库存查看
├── 记录查询
└── 我的（管理酒店/农户绑定码）

酒店端：
├── 首页（待确认 + 历史）
├── 确认页（扫码进入）
└── 我的（对账单）

农户端：
├── 首页（供货记录）
└── 我的
```

---

## 五、关键流程

**采购流程**：录入 → 选农户 → 添加品类 → 保存 → 库存增加

**销售流程**：录入 → 选酒店 → 选配送日期 → 添加品类 → 生成送货单 → 发送/打印 → 酒店确认

**异常处理**：酒店点"有异常" → 提示当面沟通 → 处理者现场改单 → 重新确认

## 七、技术要点

- 单端多角色，按 openid 路由
- 酒店/农户首次需绑定码激活
- 销售单生成小程序码，扫码确认
- 订单30分钟内可修改，留修改记录
- 数据导出 Excel 对账
